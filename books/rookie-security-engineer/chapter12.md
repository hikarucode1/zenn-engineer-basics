## 脆弱性診断とペネトレーションテストの違い

混同されがちですが、目的が異なります。

- **脆弱性診断 (Vulnerability Assessment)**: 対象システムに **どんな脆弱性があるかを網羅的に洗い出す**
- **ペネトレーションテスト (Penetration Test)**: 想定シナリオに沿って、**目標到達(例: 顧客DBの窃取)が可能か検証する**

「健康診断」と「目的を持った侵入テスト」の違いです。診断は広く浅く、ペネトレは狭く深く。

新人がまず配属されるのは脆弱性診断業務 ―― 特に **Webアプリ診断** が多いです。

## レッドチーム演習

ペネトレーションテストをさらに発展させたものが **レッドチーム演習** です。

実在する攻撃者グループの手口を模倣し、組織の **人・プロセス・技術** を総合的に試します。SOC(ブルーチーム)には事前に知らせず、「検知できるか」「対応できるか」までを試すのが特徴です。

国内ではまだ大規模に行えるベンダーは限られていますが、金融機関や大手事業会社では実施が広がっています。

## 主なツール

最初に出会うツールは次の3つです。

- **Burp Suite**: Webアプリ診断のデファクトスタンダード。HTTP通信を傍受・改ざんできる
- **Nmap**: ネットワークスキャンの定番。開いているポートやサービスを調査
- **Metasploit**: ペネトレ用のフレームワーク。多数の攻撃モジュールを内蔵

Burp Suiteには無料の Community Edition があるので、学習用には十分です。Juice Shop と組み合わせれば手を動かす環境はすぐ整います。

## バグバウンティという働き方

企業が「脆弱性を見つけたら報奨金を払います」と公募する制度が **バグバウンティ** です。

代表的なプラットフォーム: HackerOne、Bugcrowd、国内では IssueHunt や LINE Security Bug Bounty Program など。

学生のうちから取り組めば、就職活動で強力な実績になります。実際に「学生時代にバグバウンティで○件報告」という経験で大手企業に新卒入社する事例も増えています。

:::message alert
許可されたプログラム以外のシステムに攻撃を仕掛けるのは犯罪です。スコープと規約を必ず守ってください。
:::

## 倫理と契約

レッドチーム職で最も重要なのは技術力ではなく **倫理観と契約理解** です。

許可なく他社のシステムを調べると、技術的にどれだけ正しくても **不正アクセス禁止法違反** になります。診断契約書には対象範囲・期間・手法・連絡体制が細かく規定され、その通りに動くことが求められます。

「攻撃する側」でありながら、誰よりも法令とルールに敏感であること ―― これがレッドチームの専門職としての矜持です。
