## 3ステップ学習ロードマップ

未経験から最初の就職までを、おおまかに3段階で考えるのがおすすめです。

### ステップ1: ITの土台を作る(3〜6ヶ月)

- ネットワーク (TCP/IP、HTTP、DNS) の基礎
- Linux の基本操作(シェル、パーミッション、プロセス)
- プログラミング 1言語(Python推奨)
- 簡単なWeb開発(1ページでもよいので作る)

**推奨書籍**: 『マスタリングTCP/IP 入門編』『新しいLinuxの教科書』

### ステップ2: セキュリティの基礎を入れる(3〜6ヶ月)

- 本書をはじめとする入門書を読む
- OWASP Juice Shop で Web脆弱性を体験
- Wireshark でネットワーク観察
- AWS無料枠で IAM・S3 を触る
- 基本情報技術者 → 情報処理安全確保支援士 (SC) を視野に

### ステップ3: 実績を作る(継続)

- CTF に参加 (picoCTF → SECCON ジュニア → 常設プラットフォーム)
- バグバウンティに挑戦
- 学んだことを Zenn やブログで発信
- セキュリティ・ミニキャンプ、SecHack365 などのプログラムに応募

「ステップ3」が一番大事です。**手を動かして、それを言語化して残す**。これに勝るアピール材料はありません。

## 主要な資格

セキュリティ業界で評価される資格を、難易度順に整理します。

| 資格 | 主催 | レベル | 特徴 |
|------|------|--------|------|
| 基本情報技術者 | IPA | 入門 | IT全般。学生のうちに取りたい |
| 応用情報技術者 | IPA | 中級 | 一段深い知識。SCの登竜門 |
| CompTIA Security+ | CompTIA | 中級 | 国際的に通用する入門資格 |
| 情報処理安全確保支援士 (SC) | IPA | 中〜上級 | 国家資格。実務者にも人気 |
| CISSP | (ISC)² | 上級 | マネジメント色強め。実務経験が必要 |
| OSCP | Offensive Security | 上級・実技 | 実機ペネトレ。レッドチーム志望者の登竜門 |

学生のうちは **基本情報 → 応用情報 → SC** の流れが王道です。OSCPは費用が高いので就職後に挑戦する人が多いです。

## CTF の始め方

**CTF (Capture The Flag)** はセキュリティの競技です。問題を解いて「フラグ」と呼ばれる文字列を獲得することで得点します。

初心者向けの定番:

- **picoCTF** (米Carnegie Mellon主催): 教育目的の常設プラットフォーム。完全無料・英語
- **CpawCTF**: 日本語の入門向け常設CTF
- **SECCON Beginners**: 国内最大級のセキュリティイベントの初心者向け部門
- **TryHackMe / Hack The Box**: 常設のハンズオン学習プラットフォーム

最初の1問が解けたときの達成感は格別です。**「分からない → 検索 → 試す → 解ける」** のサイクルを楽しめるかで適性が分かります。

## インターン・新卒採用の探し方

セキュリティ業界の新卒採用情報の集め方:

- 各社の採用ページを直接見に行く(転職サイトには載らないこともある)
- セキュリティイベント (Code Blue、CODE BLUE、AVTOKYO、Internet Week) に参加して人脈を作る
- セキュリティ・ミニキャンプ、SecHack365 などの育成プログラムに応募
- Twitter (X) でセキュリティエンジニアをフォロー、勉強会情報をキャッチ

特に **セキュリティ・キャンプ** は国内最大級の若手育成プログラムで、修了生は就活で大きなアドバンテージを得られます。22歳以下なら一度は応募を検討してみてください。

## ポートフォリオ戦略

セキュリティ職の就活では、「**何を学んだか**」より「**何を作ったか・解いたか・書いたか**」が見られます。

おすすめのアウトプット:

- **CTFのWriteup**: 解いた問題の解説をブログにまとめる
- **脆弱性検証記事**: 公開された CVE を自分の環境で再現してみる
- **ツール自作**: 簡単なポートスキャナや診断補助ツールをGitHubに公開
- **書評**: 読んだ技術書の要約と感想

完璧でなくてかまいません。**継続して発信している** ことそのものが、最大のアピールになります。
