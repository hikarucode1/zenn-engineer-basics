## レイヤ別の脅威

ネットワークセキュリティを語るときは、**OSI参照モデル** や **TCP/IPモデル** のレイヤで整理すると見通しが良くなります。

- **物理層 / データリンク層**: ケーブル盗聴、MACアドレス偽装
- **ネットワーク層 (IP)**: IPスプーフィング、ルーティング攻撃
- **トランスポート層 (TCP/UDP)**: ポートスキャン、SYN Flood
- **アプリケーション層**: DNS汚染、HTTP系の攻撃

低いレイヤほど物理的な対策が中心になり、高いレイヤほどソフトウェア的な対策が中心になります。

## ファイアウォール / IDS/IPS / WAF の役割の違い

よく混同される3つのデバイスです。

- **ファイアウォール (FW)**: 通信を「通すか・止めるか」をルールで判定する関所
- **IDS (Intrusion Detection System)**: 不正な通信を **検知して通知** する
- **IPS (Intrusion Prevention System)**: 不正な通信を **検知して遮断** する
- **WAF (Web Application Firewall)**: Webアプリ層 (HTTP/HTTPS) の攻撃に特化した防御

イメージとしては、FWが「建物の入口の守衛」、IPSが「館内の警備員」、WAFが「Webサービスの専属ボディガード」です。それぞれ役割が違うので、多くの組織では **多層防御** として組み合わせます。

## VPN とゼロトラスト

- **VPN (Virtual Private Network)**: 公衆回線上に暗号化トンネルを張り、社内ネットワークに安全に接続する仕組み
- **ゼロトラスト**: 「社内も社外も信用しない。すべての通信を毎回検証する」という新しい考え方

長らくセキュリティの世界は「社内は安全・社外は危険」という **境界型防御** が主流でした。しかしリモートワークとクラウド利用の拡大で、もはや「社内」という境界自体が曖昧になっています。そこで「**何も信用せず、毎回認証・認可する**」というゼロトラストへ徐々に移行しているのが現代の潮流です。

## TLS の基本

HTTPS の "S" を実現している暗号化プロトコルが **TLS (Transport Layer Security)** です。古いSSLの後継で、現代のWeb通信のほぼすべてに使われています。

TLSが提供するのは次の3つです。

- **暗号化**: 第三者に通信内容を見られない
- **完全性**: 通信が改ざんされていないことを保証
- **真正性**: 接続先が本物のサーバーであることを証明書で保証

詳細な暗号アルゴリズムは後回しでよいですが、「鍵交換 → 共通鍵で暗号化通信」という流れは早めに頭に入れておくと、後々の暗号学習がスムーズです。

## やってみよう: Wiresharkで自分の通信を覗く

**Wireshark** はパケットキャプチャの定番ツールです。自宅PCにインストールして、自分のWeb閲覧時の通信を眺めてみてください。

HTTPSで暗号化された通信は中身が読めませんが、TCPの3-wayハンドシェイクやTLSのハンドシェイクの様子は観察できます。教科書で読むのとは比較にならない理解が得られます。

:::message alert
他人の通信を許可なく傍受するのは違法です。必ず自分の機器・自分の通信で行ってください。
:::
