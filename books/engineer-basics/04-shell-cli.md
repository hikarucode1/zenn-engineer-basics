## この章のゴール

「黒い画面」を見て「うっ」となるエンジニア向けの章です。
新人時代に一番ハードルが高いのが **CLI (コマンドラインインターフェース)** ですが、慣れると **GUI より圧倒的に速く、正確で、再現性がある** 道具になります。

この章では、OS と対話するためのシェル — bash、zsh、PowerShell — の世界を覗きます。

## 4-1. シェルとは何か

**シェル (shell)** は、あなたの入力した文字列を OS へのコマンドに変換するプログラムです。「殻」という名のとおり、OS のカーネルを包む外側の層。

- **macOS / Linux**: bash, zsh, fish などが主流
- **Windows**: PowerShell, cmd, Windows Terminal

ターミナル (黒い画面) は単に「シェルを動かす窓」であって、シェル自体ではありません。

## 4-2. 必須コマンド10個

これさえ覚えれば日常作業の8割は回ります。

| コマンド | 役割 | 例 |
|---------|------|----|
| `pwd` | 今どこにいるか | `pwd` |
| `ls` | ファイル一覧 | `ls -la` |
| `cd` | ディレクトリ移動 | `cd ~/dev` |
| `mkdir` | フォルダ作成 | `mkdir new-app` |
| `cp` | コピー | `cp a.txt b.txt` |
| `mv` | 移動/リネーム | `mv old.txt new.txt` |
| `rm` | 削除 | `rm file.txt` |
| `cat` | 中身表示 | `cat config.json` |
| `grep` | 検索 | `grep "error" log.txt` |
| `man` | マニュアル | `man ls` |

:::message alert
`rm -rf /` は **絶対に実行しない** こと。OS をまるごと消します。`rm` は ゴミ箱を経由しません。
:::

## 4-3. パス — ファイルの住所

ファイルやフォルダの場所を表す文字列が **パス (path)** です。

- **絶対パス**: ルート (`/`) から書く。`/Users/hikaru/memo.txt`
- **相対パス**: 今いる場所からの相対。`./memo.txt`、`../parent/file.txt`

特別な記号:

| 記号 | 意味 |
|------|------|
| `.` | カレントディレクトリ (今いる場所) |
| `..` | 一つ上 |
| `~` | ホームディレクトリ (`/Users/あなた`) |
| `/` | ルート |

## 4-4. 標準入出力とリダイレクト

シェルの真髄は **入力 (stdin) → 処理 → 出力 (stdout) / エラー出力 (stderr)** という流れを自由に組み替えられることです。

```bash
# 出力をファイルに書き出す
ls > files.txt

# 追記
ls >> files.txt

# エラー出力もまとめる
command > out.log 2>&1

# ファイルを入力として渡す
sort < unsorted.txt
```

3つのストリーム:

| 名前 | 番号 | 役割 |
|------|------|------|
| stdin | 0 | 標準入力 |
| stdout | 1 | 標準出力 (普通の結果) |
| stderr | 2 | 標準エラー出力 (エラーメッセージ) |

## 4-5. パイプ — コマンドをつなぐ

`|` (パイプ) を使うと、**前のコマンドの出力を次のコマンドの入力に渡せます**。

```bash
# プロセス一覧から chrome だけ抽出
ps aux | grep chrome

# ファイル数を数える
ls | wc -l

# ログから ERROR を含む行を最新10件
cat app.log | grep ERROR | tail -10
```

「小さなコマンドを組み合わせて大きな仕事をする」 — これが UNIX 哲学の中心です。

## 4-6. 環境変数

シェル上の **設定値の入れ物** が環境変数です。アプリは環境変数を読んで挙動を変えます。

```bash
# 設定
export API_KEY="sk-xxx"

# 確認
echo $API_KEY

# 一覧
env
```

代表的な環境変数:

| 変数 | 意味 |
|------|------|
| `PATH` | コマンドを検索するディレクトリ一覧 |
| `HOME` | ホームディレクトリ |
| `USER` | ログインユーザー名 |
| `LANG` | 言語設定 |

:::message alert
**API キーや DB パスワードはコードに書かず環境変数に入れる**。GitHub にうっかり commit すると数分でクローラに収集されます。第17章のセキュリティで詳しく扱います。
:::

## 4-7. シェルスクリプト

コマンドの並びをファイルに書いて再利用できるようにしたのが **シェルスクリプト** です。`.sh` 拡張子で保存し、`bash script.sh` で実行します。

```bash
#!/bin/bash
# バックアップを取って ZIP にまとめる例

DATE=$(date +%Y%m%d)
mkdir -p backups/
zip -r "backups/backup-${DATE}.zip" src/
echo "Backup created: backup-${DATE}.zip"
```

毎日繰り返す手作業はスクリプトに置き換えると、**ミスらず、速く、後輩にも引き継げる** 資産になります。

## 4-8. キーボード操作のショートカット

ターミナルでは Emacs 流のキーバインドが標準で効きます。

| キー | 動作 |
|------|------|
| `Ctrl + A` | 行頭へ |
| `Ctrl + E` | 行末へ |
| `Ctrl + U` | 行頭まで削除 |
| `Ctrl + K` | 行末まで削除 |
| `Ctrl + R` | 履歴検索 |
| `Tab` | 補完 |
| `Ctrl + C` | 実行中のプロセスを止める |
| `Ctrl + D` | EOF (シェルから抜ける) |

`Ctrl + R` で履歴検索 → タイプ補完、を覚えるだけで作業速度が倍になります。

## 4-9. 同じことを GUI でやらない理由

CLI を勧める理由は3つ:

1. **再現性**: 同じコマンドは同じ結果を返す。スクショより共有しやすい
2. **自動化しやすい**: スクリプト化・CI でそのまま動く
3. **SSH 越しに使える**: リモートサーバーは GUI を持たないことが多い

GUI が悪いわけではありません。**「2回以上やる作業は CLI で書く」** がエンジニアの基本姿勢です。

## まとめ

- **シェル** は OS と対話するための言語。bash / zsh / PowerShell が代表
- 必須コマンド `pwd / ls / cd / mkdir / cp / mv / rm / cat / grep / man` を体に入れる
- パスは絶対 (`/...`) と相対 (`./`, `../`, `~`) を使い分ける
- **stdin / stdout / stderr** をリダイレクト (`>`、`<`、`2>&1`) で操作
- **パイプ (`|`)** で小さなコマンドをつなげるのが UNIX 哲学の核
- 環境変数で設定を渡す。**秘密情報は環境変数で**、コードに書かない
- 繰り返し作業は **シェルスクリプト** にして資産化

次の章では、いよいよ「プログラミングそのもの」 — 言語、コンパイル、実行の仕組みに入ります。
